Tentativa de Lighthouse (automática, sessão atual)

Comando executado contra o preview local (porta pode variar; exemplo http://127.0.0.1:4174/app/guide): npx lighthouse … --output-path=public/lighthouse-report-guide-mobile.html --chrome-flags="--headless --no-sandbox".

Bloqueio no Windows: o Lighthouse encerrou com EBUSY: resource busy or locked, unlink …\Temp\lighthouse.*\Default\Account Web Data (perfil temporário do Chromium em uso ou bloqueado por outra instância). Possíveis mitigações: fechar janelas do Chrome/Chromium Edge, fixar uma pasta limpa via --user-data-dir, ou rodar o mesmo comando no WSL. Sem HTML gerado pelo Lighthouse nesta máquina.

Comando sugerido (Windows / PowerShell)

Ajuste porta se o preview usar outra. Substitua a URL pela do preview com hash, se usar âncoras.

cd "C:\Users\PC\Documents\VS CODE\febracis-dre"
npm run build
npm run preview
# outro terminal — após login manual no navegador, cópia de cookie opcional --
npx lighthouse http://localhost:4173/app/guide ^
  --only-categories=accessibility,performance,best-practices,seo ^
  --preset=desktop ^
  --screenEmulation.disabled ^
  --output=html ^
  --output-path=./public/lighthouse-report-guide-desktop.html
npx lighthouse http://localhost:4173/app/guide ^
  --only-categories=accessibility,performance,best-practices,seo ^
  --form-factor=mobile ^
  --throttling-method=simulate ^
  --output=html ^
  --output-path=./public/lighthouse-report-guide-mobile.html

Se o Chrome não estiver instalado ou o comando falhar com erro de navegador, instale Chromium ou use variável de ambiente CHROME_PATH conforme a documentação do Lighthouse.
